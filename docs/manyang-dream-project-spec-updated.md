# 마냥 꿈해몽 프로젝트 기획서

> **고양이 해몽 백과사전을 기반으로 꿈속 상징을 읽고, 사용자의 꿈을 카드와 기록으로 남겨주는 감성 꿈 리딩 서비스**

---

## 문서 업데이트 내역

- 프로젝트명을 **마냥 꿈해몽**으로 확정한다.
- “달빛 냥몽관”은 최종 프로젝트명이 아니라, 서비스의 공간적/비주얼 콘셉트로만 활용한다.
- 최근 확장된 UI 기획을 반영해 상징 백과, 꿈 달력, 아침 기분 기록, 자기 전 꿈 씨앗, 설정/마이페이지 화면을 추가한다.
- 꿈 카드 외에 공유형 결과물인 **꿈 영수증(dream receipt)** 포맷을 추가한다.
- MVP 우선순위를 “꿈 입력 → 해몽 결과 → 기록/달력 → 기억나지 않는 날의 기록” 중심으로 재정렬한다.
- 제작이 필요한 UI 화면과 비주얼 에셋 체크리스트를 별도 섹션으로 정리한다.

---

## 1. 프로젝트 개요

### 1.1 프로젝트 이름

**마냥 꿈해몽**

프로젝트명은 **마냥 꿈해몽**으로 확정한다.  
사용자가 “마냥” 편하게 꿈을 말하면, 고양이 해몽사가 꿈속 상징을 찾아 “꿈해몽”으로 풀어주는 서비스라는 의미를 가진다.

브랜드 사용 원칙:

- 최종 서비스명: **마냥 꿈해몽**
- 앱/문서/발표 자료의 기본 표기: **마냥 꿈해몽**
- “달빛 냥몽관”은 서비스 안의 공간 콘셉트 또는 비주얼 테마명으로만 사용
- 영문명은 MVP 단계에서는 확정하지 않고, 필요 시 보조 브랜드명으로 검토

후보 영문명:

- Moonlit Meowmancy
- Cat Dream Archive
- Meow Dream Oracle
- Neko Dream Library

### 1.2 한 줄 소개

> 어젯밤 꿈을 고양이 해몽사가 읽고, 꿈속 상징을 백과사전과 연결해 해석한 뒤, 하나의 꿈 카드로 남겨주는 감성 꿈 리딩 서비스.

### 1.3 핵심 컨셉

마냥 꿈해몽은 단순히 꿈을 입력하면 AI가 즉흥적으로 해몽해주는 서비스가 아니다.  
이 프로젝트의 핵심은 **꿈속에 등장한 상징을 고양이 해몽 백과사전과 연결하고, 그 해석을 감성적인 카드와 기록으로 축적하는 것**이다.

즉, 사용자는 매일 자신의 꿈을 입력하고, 서비스는 다음과 같은 과정을 거친다.

```text
꿈 입력
↓
꿈속 상징 추출
↓
꿈 해몽 백과사전 검색
↓
고양이 해몽사의 리딩 생성
↓
오늘의 작은 처방 제공
↓
꿈 카드 생성
↓
꿈 기록과 상징 히스토리에 저장
```

### 1.4 서비스 정체성

이 서비스는 다음 세 가지의 결합이다.

1. **AI 꿈 리딩 서비스**  
   LLM을 활용해 사용자의 꿈을 구조화하고 해석한다.

2. **꿈 해몽 백과사전**  
   꿈속 상징을 해석하는 기준이 되는 자체 지식 베이스를 제공한다.

3. **감성 아카이브 / 카드 수집 서비스**  
   꿈 해석 결과를 카드 형태로 저장하고, 사용자의 꿈 기록을 시각적으로 쌓아간다.

## 2. 왜 이 아이디어인가?

### 2.1 기존 꿈해몽 서비스의 문제

기존 꿈해몽 서비스는 대체로 다음과 같은 한계를 가진다.

| 문제 | 설명 |
|---|---|
| 너무 단정적임 | “이빨 빠지는 꿈 = 흉몽”처럼 정답처럼 말함 |
| 오래된 운세 사이트 느낌 | 디자인과 언어가 낡고 신뢰감이 낮음 |
| 개인 맥락 부족 | 사용자의 실제 꿈 문맥을 제대로 반영하지 않음 |
| 결과가 휘발됨 | 해석을 보고 끝나며 기록이나 수집 구조가 약함 |
| AI 서비스와 차별화 부족 | ChatGPT에 물어보는 것과 다를 바 없어 보일 수 있음 |

### 2.2 마냥 꿈해몽의 차별점

마냥 꿈해몽은 단순한 꿈해몽기가 아니라, **꿈속 상징을 백과사전과 연결해 읽어주는 고양이 해몽관**이다.

차별화 포인트는 다음과 같다.

| 차별화 요소 | 설명 |
|---|---|
| 꿈 해몽 백과사전 기반 | LLM이 아무렇게나 해석하지 않고, 자체 상징 사전을 참고함 |
| 고양이 해몽사 캐릭터 | 결과를 딱딱하게 설명하지 않고 감성적이고 부드럽게 전달함 |
| 꿈 카드 시스템 | 해몽 결과를 카드로 저장해 수집성과 재방문 이유를 만듦 |
| 상징 히스토리 | 사용자의 꿈에 자주 등장한 상징을 누적해 보여줌 |
| 작은 처방 | 운세 대신 오늘 해볼 수 있는 작은 행동을 제안함 |
| 감성적 UI | 달빛, 꿈, 카드, 고양이, 백과사전이라는 시각 콘셉트가 명확함 |

### 2.3 핵심 문장

> 사라지는 꿈을 고양이가 읽고, 상징과 카드로 남겨준다.

## 3. 타깃 사용자

### 3.1 1차 타깃

- 꿈을 자주 기억하거나 꿈에 의미를 부여하는 사람
- 감성적인 기록 앱을 좋아하는 사람
- 타로, 운세, 심리테스트, 다이어리 앱을 가볍게 즐기는 사람
- 귀여운 캐릭터 기반 서비스를 좋아하는 사람
- 자기 감정과 무의식적 패턴을 부드럽게 돌아보고 싶은 사람

### 3.2 2차 타깃

- AI 기반 감성 서비스를 좋아하는 사용자
- SNS에 예쁜 카드 결과를 공유하고 싶은 사용자
- 일기 앱은 부담스럽지만 짧은 기록은 남기고 싶은 사용자
- 고양이 캐릭터 IP를 좋아하는 사용자

### 3.3 사용자가 이 서비스를 쓰는 이유

사용자는 다음 이유로 서비스를 사용할 수 있다.

1. 어젯밤 꿈이 무슨 뜻인지 궁금해서
2. 꿈을 그냥 잊어버리지 않고 예쁘게 저장하고 싶어서
3. AI가 내 꿈을 어떻게 해석하는지 재미있어서
4. 고양이 해몽사와 꿈 카드가 귀여워서
5. 내 꿈에 자주 나오는 상징을 보고 싶어서
6. 매일 아침 가볍게 나를 돌아보는 루틴이 필요해서

## 4. 핵심 사용자 경험

### 4.1 기본 사용 흐름

```text
1. 사용자가 접속한다.
2. 고양이 해몽사가 꿈을 물어본다.
3. 사용자가 어젯밤 꿈을 입력한다.
4. AI가 꿈속 상징을 추출한다.
5. 추출된 상징을 꿈 해몽 백과사전에서 검색한다.
6. 고양이 해몽사가 백과사전 내용을 참고해 해몽한다.
7. 오늘의 작은 처방을 제안한다.
8. 꿈 카드가 생성된다.
9. 사용자는 카드를 저장하거나 공유한다.
10. 꿈 기록이 아카이브에 쌓인다.
```

### 4.2 꿈을 기억하지 못한 날의 흐름

꿈 서비스에서 중요한 점은 사용자가 매일 꿈을 기억하지는 못한다는 것이다.  
따라서 “꿈을 기억하지 못함”도 하나의 정상적인 사용 흐름으로 설계해야 한다.

```text
1. 사용자가 접속한다.
2. “오늘은 꿈이 기억나지 않아요”를 선택한다.
3. 고양이가 아침 기분을 묻는다.
4. 사용자는 기분, 색, 몸 상태, 생각나는 단어 하나를 선택한다.
5. 서비스는 “사라진 꿈의 발자국 카드”를 생성한다.
6. 꿈 없음 상태도 기록에 남긴다.
```

예시 메시지:

```text
오늘 꿈은 발자국만 남기고 사라졌다냥.
그래도 아침의 기분은 기록해둘 수 있다냥.
```

이 흐름은 매일 접속을 유도하는 데 중요하다.  
꿈을 기억하지 못한 날에도 앱을 열 이유가 생긴다.

## 5.1 꿈 입력 기능

### 목적

사용자가 기억나는 꿈을 자유롭게 입력할 수 있게 한다.

### 입력 요소

필수:

- 꿈 내용 텍스트

선택:

- 깼을 때 기분
- 꿈의 분위기
- 꿈에서 가장 기억나는 장면
- 꿈에 등장한 사람/동물/장소
- 꿈을 꾼 날짜

### 예시 UI 문구

```text
어젯밤 꿈을 들려달라냥.
짧아도 괜찮고, 뒤죽박죽이어도 괜찮다냥.
```

### 입력 예시

```text
낡은 학교 복도에서 계속 교실을 찾고 있었는데,
신발이 없어져서 맨발로 뛰어다니는 꿈을 꿨어.
```

## 5.3 꿈 해몽 백과사전

### 목적

꿈속 상징에 대한 해석 기준을 제공한다.

LLM이 즉흥적으로 해몽하지 않도록, 서비스 내부의 꿈 해몽 백과사전을 참고하게 한다.  
이 백과사전은 “정답 사전”이 아니라 “해석 재료 라이브러리”로 사용한다.

### 중요한 원칙

피해야 할 방식:

```text
이빨 빠지는 꿈은 가족에게 안 좋은 일이 생긴다는 뜻입니다.
```

지향할 방식:

```text
이빨은 상실감, 자기 이미지, 말하지 못한 것, 통제감 약화와 연결될 수 있다.
꿈의 분위기와 함께 해석해야 한다.
```

## 5.4 고양이 해몽사 시스템

### 목적

해몽 결과를 단순한 AI 답변이 아니라, 캐릭터가 읽어주는 감성 경험으로 만든다.

### 고양이 해몽사 예시

| 해몽사 | 성격 | 해석 스타일 |
|---|---|---|
| 검은냥 | 신비롭고 조용함 | 상징적이고 깊은 해석 |
| 흰냥 | 다정하고 부드러움 | 감정 위로 중심 |
| 치즈냥 | 현실적이고 가벼움 | 실용적 조언 중심 |
| 삼색냥 | 엉뚱하고 창의적 | 상상력과 유머 중심 |
| 회색냥 | 분석적이고 차분함 | 반복 패턴과 구조 중심 |

### MVP 추천

MVP에서는 처음부터 고양이를 많이 넣기보다, 1~2마리만 제공하는 것이 좋다.

추천 MVP 캐릭터:

1. **검은냥**  
   대표 고양이. 달빛, 꿈, 상징, 해몽 분위기와 가장 잘 맞음.

2. **흰냥**  
   다정한 위로형. 해석 결과가 너무 무겁게 느껴지지 않도록 보완함.

### 예시 말투

검은냥:

```text
이 꿈은 사라진 것이 아니라 아직 드러나지 않은 것을 말한다냥.
```

흰냥:

```text
이 꿈은 불안보다 피로에 가까워 보인다냥.
오늘은 속도를 조금 낮춰도 괜찮다냥.
```

치즈냥:

```text
요즘 할 일이 너무 많아서 머릿속이 복도처럼 꼬인 것 같다냥.
```

## 5.5 꿈 해석 결과

### 결과 화면 구성

해석 결과는 다음 구조로 제공한다.

```text
1. 꿈 요약
2. 꿈속 상징
3. 백과사전 기반 해석
4. 고양이 해몽사의 리딩
5. 오늘의 작은 처방
6. 꿈 카드
```

### 예시 결과

입력:

```text
낡은 학교에서 신발을 잃어버리고 계속 복도를 뛰어다니는 꿈을 꿨어.
```

출력:

```text
꿈 요약
낡은 학교 복도에서 교실을 찾다가 신발을 잃어버리고 맨발로 뛰어다닌 꿈.

꿈속 상징
- 학교: 평가, 준비, 과거의 기준
- 복도: 전환 구간, 선택 전 상태
- 신발: 이동 준비, 기반, 방향성
- 뛰어다님: 조급함, 압박감

고양이 해몽
이 꿈은 단순히 무언가를 잃어버린 꿈이라기보다,
어디론가 가야 한다는 압박은 있는데 아직 발을 디딜 준비가 충분하지 않다고 느끼는 장면에 가깝다냥.

특히 신발은 앞으로 나아가기 위한 기반과 관련되고,
복도는 아직 목적지에 도착하기 전의 전환 구간을 뜻할 수 있다냥.

오늘의 작은 처방
새로운 일을 시작하기 전에, 지금 가장 불안한 준비물 하나를 먼저 확인해보라냥.

꿈 카드
「맨발의 복도냥」
키워드: 준비 / 방향 / 조급함
```

## 5.6 오늘의 작은 처방

### 목적

운세 대신 사용자의 하루에 작게 연결되는 행동 제안을 제공한다.

### 원칙

- 너무 거창하면 안 된다.
- 심리 치료처럼 말하면 안 된다.
- 사용자가 바로 할 수 있는 작은 행동이어야 한다.
- 꿈 해석과 연결되어야 한다.

### 예시

```text
책상 위에서 가장 오래 방치된 물건 하나를 치우기.
```

```text
오늘 할 일 중 가장 작은 것 하나만 먼저 끝내기.
```

```text
오래 미룬 메시지 하나에 짧게 답장하기.
```

```text
잠들기 전에 내일 걱정 하나를 종이에 적어두기.
```

### 표현 방식

“오늘의 운세” 대신 다음 표현을 사용한다.

| 흔한 표현 | 마냥 꿈해몽식 표현 |
|---|---|
| 오늘의 운세 | 오늘의 작은 처방 |
| 행운의 색 | 오늘의 빛깔 |
| 행운의 물건 | 곁에 두면 좋은 것 |
| 조심할 것 | 오늘 천천히 볼 것 |
| 타로 카드 | 꿈 카드 |
| 점괘 | 고양이의 해석 |

## 5.7 꿈 카드 시스템

### 목적

해몽 결과를 저장 가능하고 공유 가능한 결과물로 만든다.

### 카드 구성

```text
카드명
고양이 일러스트 또는 아이콘
꿈 키워드
꿈속 주요 상징
짧은 해석
오늘의 작은 처방
해몽사 이름
생성 날짜
```

### 예시 카드

```text
카드명: 맨발의 복도냥
속성: 전환 / 준비 / 조급함
상징: 학교, 복도, 신발
```

- 그믐달 카드

- 비의 카드

## 5.9 백과사전 열람 기능

### 목적

사용자가 꿈을 입력하지 않아도 꿈 상징을 탐색할 수 있게 한다.

### 기능

- 상징 검색
- 카테고리별 탐색
- 관련 상징 보기
- 내 꿈에 등장한 횟수 표시
- 백과사전 항목에서 관련 꿈 기록으로 이동

### 예시

```text
문

고양이에게 문은 늘 수상한 존재다냥.
열려 있으면 들어갈지 말지 고민하게 만들고,
닫혀 있으면 괜히 더 들어가고 싶게 만든다냥.

꿈에서 문은 보통 경계, 선택, 기회, 망설임과 연결될 수 있다냥.
문이 열렸는지, 잠겼는지, 누가 문 너머에 있었는지가 중요하다냥.

관련 상징:
열쇠, 방, 복도, 집, 창문

내 꿈에 등장한 횟수:
3회
```

## 5.10 상징 히스토리

### 목적

개인 사전처럼 부담스럽게 만들기보다, 공통 백과사전 위에 사용자의 등장 기록을 얹는다.

### 기능

- 내 꿈에 자주 등장한 상징
- 최근 30일 상징 변화
- 감정과 함께 자주 등장한 상징
- 특정 상징이 등장한 꿈 목록

### 예시

```text
최근 자주 열린 백과사전 항목

1. 복도 — 6회
2. 신발 — 4회
3. 엘리베이터 — 3회
4. 비 — 3회
5. 문 — 3회
```

고양이 코멘트:

```text
요즘 도민님의 꿈에서는 ‘복도’가 자주 보인다냥.
아직 목적지에 도착하기 전의 장면들이 반복되는 것 같다냥.
```

## 5.11 꿈 영수증 시스템

### 목적

꿈 해석 결과를 짧고 감성적인 공유 이미지로 제공한다.  
꿈 카드가 사용자의 꿈을 수집하는 저장형 결과물이라면, 꿈 영수증은 하루의 꿈을 짧게 요약해 보여주는 공유형 결과물이다.

### 꿈 카드와의 차이

| 구분 | 꿈 카드 | 꿈 영수증 |
|---|---|---|
| 목적 | 저장과 수집 | 요약과 공유 |
| 형태 | 카드형 컬렉션 | 세로형 영수증 이미지 |
| 사용 위치 | 결과 화면, 기록, 카드함 | 결과 화면, 공유, 저장 이미지 |
| 분위기 | 타로 카드, 꿈 카드 | 오래된 종이, 꿈 기록표 |

### 구성 요소

```text
dream receipt

date
dream summary
symbols
mood
cat reading
small prescription
cat reader
```

### 예시

```text
dream receipt

date: 2026.05.24
dream: 끝없는 복도를 걷는 꿈
symbols: 복도 / 문 / 어둠
mood: 조급함

cat reading:
아직 도착하지 않은 마음이 조용히 길을 찾고 있다냥.

small prescription:
오늘은 결정하지 못한 일 하나를 종이에 적어보기.
```

### MVP 적용 방식

1차 MVP에서는 이미지 생성 모델을 사용하지 않고, 고정된 영수증 템플릿에 텍스트를 렌더링하는 방식으로 구현한다.  
이후 고급 테마, 고해상도 저장, 월간 꿈 영수증 모음 PDF 등으로 확장할 수 있다.

## 6. 리텐션 설계

## 6.1 매일 접속 루프

꿈 서비스는 아침 루틴과 잘 맞는다.

### 아침 루프

```text
1. 꿈이 기억나는지 확인
2. 기억나면 꿈 입력
3. 기억나지 않으면 아침 기분 기록
4. 오늘의 작은 처방 확인
5. 꿈 카드 또는 사라진 꿈 카드 저장
```

### 밤 루프

```text
1. 자기 전 접속
2. 오늘 밤 꿈에게 묻고 싶은 질문 선택
3. 꿈 씨앗 심기
4. 다음날 아침 꿈 기록 유도
```

### 꿈 씨앗 예시

```text
오늘 밤 꿈에게 묻고 싶은 게 있나요?

- 지금 내 마음이 궁금해
- 프로젝트에 대한 힌트가 필요해
- 그냥 이상한 꿈을 보고 싶어
- 아무것도 묻지 않고 잠들래
```

## 6.2 꿈을 기억하지 못하는 날의 리텐션

꿈을 기억하지 못한 날에도 서비스를 쓸 수 있게 한다.

### 대체 입력

- 오늘 아침 기분
- 떠오르는 색
- 몸 상태
- 오늘 가장 먼저 든 생각
- 꿈이 남긴 느낌

### 결과

- 사라진 꿈의 발자국 카드
- 오늘의 작은 처방
- 고양이 한마디

### 목적

사용자가 다시 돌아와 자신의 꿈 패턴을 확인하게 만든다.

```text
이번 주 도민님의 꿈 날씨

자주 등장한 장소:
- 학교
- 복도

「닫히지 않는 문 앞의 흰냥」

고양이 해석:
이번 주의 꿈들은 무언가를 시작하고 싶지만,
아직 정확한 입구를 찾지 못한 상태를 자주 보여줬다냥.

다음 주 작은 처방:
새로운 일을 벌리기보다, 이미 열린 문 하나를 끝까지 밀어보기.
```

## 7. 유료화 전략

## 7.1 무료 플랜

무료 플랜은 서비스의 핵심 재미를 느낄 수 있어야 한다.

제공 기능:

- 하루 1회 꿈 리딩
- 기본 고양이 해몽사 1마리
- 기본 꿈 카드 생성
- 최근 7개 꿈 저장
- 기본 백과사전 항목 열람
- 기본 상징 추출

## 7.2 유료 플랜: Moon Pass

유료 플랜은 “더 많은 해몽”보다 **더 깊은 아카이브와 더 예쁜 결과물**에 초점을 둔다.

제공 기능:

- 무제한 꿈 아카이브
- 심화 꿈 해몽 백과사전 열람
- 내 꿈에 자주 등장한 백과사전 항목 분석
- 주간/월간 꿈 리포트
- 고양이 해몽사 추가
- 고급 꿈 카드 테마
- 카드 이미지 저장
- 꿈 카드북 PDF 내보내기
- 검색/필터 고급 기능
- 프라이버시 잠금 기능

## 7.3 일회성 결제 아이템

구독 외에 소장형 아이템을 둘 수 있다.

- 특별 카드 테마
- 커스텀 고양이 해몽사
- 월간 꿈 카드북 PDF
- 카드 이미지 고해상도 저장팩
- 고양이 해몽사 의상/방 꾸미기

## 7.4 사용자가 돈을 내는 이유

사용자가 결제하는 이유는 다음이어야 한다.

```text
내 꿈이 그냥 사라지지 않고,
고양이 해몽관 안에 카드와 상징 기록으로 쌓이기 때문에.
```

더 구체적으로는:

- 내 꿈을 오래 저장하고 싶다.
- 꿈 카드가 예뻐서 소장하고 싶다.
- 자주 등장하는 상징을 보고 싶다.
- 주간/월간 꿈 리포트를 받고 싶다.
- 더 다양한 고양이 해몽사를 만나고 싶다.
- 내 꿈을 카드북으로 만들고 싶다.

## 9. DB 설계 초안

## 9.1 users

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE,
  display_name TEXT,
  plan TEXT DEFAULT 'free',
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);
```

## 9.2 dream_entries

사용자의 꿈 원문을 저장한다.

```sql
CREATE TABLE dream_entries (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  dream_date DATE NOT NULL,
  raw_text TEXT,
  remembered BOOLEAN DEFAULT true,
  wake_mood TEXT,
  dream_mood TEXT,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);
```

## 9.3 dream_analyses

LLM 분석 결과를 저장한다.

```sql
CREATE TABLE dream_analyses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  dream_id UUID REFERENCES dream_entries(id),
  summary TEXT,
  symbols JSONB,
  emotions JSONB,
  themes JSONB,
  encyclopedia_refs JSONB,
  interpretation TEXT,
  small_prescription TEXT,
  cat_reader_type TEXT,
  model_name TEXT,
  created_at TIMESTAMP DEFAULT now()
);
```

## 9.4 dream_encyclopedia

꿈 해몽 백과사전 항목을 저장한다.

```sql
CREATE TABLE dream_encyclopedia (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  symbol TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  category TEXT NOT NULL,
  core_meanings JSONB,
  positive_readings JSONB,
  negative_readings JSONB,
  context_questions JSONB,
  related_symbols JSONB,
  cat_interpretation_hint TEXT,
  body TEXT,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);
```

## 9.5 dream_cards

생성된 꿈 카드를 저장한다.

```sql
CREATE TABLE dream_cards (

  card_type TEXT,
```

## 9.6 user_symbol_history

사용자별 상징 등장 기록을 저장한다.

## 10. LLM 파이프라인

## 10.1 기본 파이프라인

```text
1. 사용자가 꿈 입력
2. LLM이 구조화 분석 JSON 생성
3. 분석된 symbols를 기반으로 백과사전 검색
4. 관련 백과사전 항목을 LLM에 전달
5. LLM이 최종 해몽 생성
6. LLM이 꿈 카드 정보 생성
7. DB에 저장
8. 결과 화면 렌더링
```

## 10.2 1차 LLM: 꿈 구조화 분석

목적:

- 꿈 원문을 요약한다.
- 상징, 감정, 테마를 추출한다.
- 백과사전 검색에 사용할 키워드를 만든다.

출력 형식:

```json
{

  "symbols": ["string"],
  "symbol_candidates": [

      "text": "string",

      "importance": 1
    }
  ],
  "emotions": ["string"],
  "themes": ["string"],
  "dream_mood": "string",
  "safety_note": "string | null"
}
```

## 10.3 백과사전 검색

검색 방식 MVP:

- 추출된 symbols와 dream_encyclopedia.symbol을 문자열 매칭

확장 방식:

- 백과사전 항목 임베딩 저장
- 꿈 요약/상징을 임베딩해 유사 항목 검색

## 11.1 꿈 구조화 분석 프롬프트

```text
너는 꿈 텍스트를 분석해 구조화하는 분석기다.
사용자의 꿈을 심리적으로 단정하거나 진단하지 말고,
꿈에 등장한 장소, 사물, 행동, 감정, 반복 테마 후보만 추출한다.

반드시 JSON으로만 답한다.

분석 기준:
- symbols: 꿈속에서 해석에 쓸 수 있는 명사/행동/장면
- emotions: 꿈에서 느껴지는 감정 후보
- themes: 방향 탐색, 상실, 변화, 회피, 기다림 등 추상 테마
- 절대 “이 꿈은 반드시 ~이다”라고 단정하지 않는다.

사용자 꿈:
{{dream_text}}
```

## 11.2 백과사전 기반 해몽 프롬프트

```text
너는 '마냥 꿈해몽'의 고양이 해몽사다.
사용자의 꿈을 해석하되, 아래 꿈 해몽 백과사전 항목을 반드시 참고한다.

중요한 규칙:
- 진단처럼 말하지 않는다.
- 불길한 예언처럼 말하지 않는다.
- “~일 수 있다”, “~와 연결되어 보인다”처럼 가능성 기반으로 말한다.
- 사용자의 하루에 작게 도움이 되는 행동 제안을 하나 제공한다.
- 말투는 부드럽고 감성적이며, 고양이 해몽사 캐릭터를 살린다.
- 과도한 ‘냥’ 사용은 피하고, 문장 끝에 가끔만 사용한다.

사용자 꿈:
{{dream_text}}

구조화 분석:
{{analysis_json}}

{{encyclopedia_entries}}

고양이 해몽사:
{{cat_reader_profile}}

출력은 다음 JSON 형식을 따른다:
{
  "dream_reading": "...",
  "symbol_readings": [...],
  "small_prescription": "...",
  "card": {
    "name": "...",
    "type": "...",
    "keywords": [...],
    "summary": "...",
    "message": "...",
    "visual_prompt": "..."
  }
}
```

```text
상징 상세 페이지

```

## 13.7 상징 백과 전체 화면

### 목적

사용자가 꿈을 입력하지 않아도 꿈속 상징을 탐색할 수 있게 한다.  
마냥 꿈해몽이 단순 AI 해몽기가 아니라, 자체 백과사전을 가진 서비스라는 점을 보여주는 핵심 화면이다.

### 구성

```text
상단 검색창
카테고리 탭: 장소 / 사물 / 행동 / 자연 / 존재
자주 본 상징
상징 카드 리스트
각 상징의 짧은 의미, 관련 키워드, 내 꿈 등장 횟수
```

### UX 포인트

- 상징 카드는 너무 학술적으로 보이지 않게 짧고 감성적으로 구성한다.
- 사용자가 이미 기록한 꿈에 등장한 상징은 “내 꿈에 나온 상징” 표시를 붙인다.
- 백과사전은 해몽의 근거가 되는 공간이므로, 결과 화면에서도 자연스럽게 연결한다.

## 13.8 상징 백과 상세 화면

### 목적

하나의 상징을 더 깊게 읽을 수 있게 한다.  
예를 들어 “복도”를 선택하면, 복도가 가진 핵심 의미와 관련 상징, 사용자의 꿈 기록을 함께 보여준다.

### 구성

```text
상징명
고양이 한마디
핵심 의미
긍정적 해석 방향
부정적 해석 방향
꿈에서 확인할 질문
관련 상징
내 꿈에 등장한 기록
이 상징이 나온 꿈 카드 리스트
```

### 예시: 복도

```text
복도

고양이 한마디:
복도는 아직 도착하지 않은 마음이 걷는 길이다냥.

핵심 의미:
전환 구간, 선택 전 상태, 목적지 탐색, 이동 중인 마음

확인할 질문:
- 복도 끝에는 무엇이 있었나요?
- 문이 열려 있었나요, 닫혀 있었나요?
- 혼자 걷고 있었나요?
- 급하게 뛰고 있었나요, 천천히 걷고 있었나요?
```

## 13.9 기록 / 꿈 달력 화면

### 목적

사용자의 꿈 기록을 날짜 기반으로 다시 볼 수 있게 한다.  
꿈 카드가 하나의 결과물이라면, 꿈 달력은 사용자의 꿈 기록이 쌓이는 아카이브 화면이다.

### 구성

```text
월간 꿈 달력
꿈을 기록한 날 표시
꿈을 기억하지 못한 날 표시
대표 꿈 카드
감정/상징 요약
하단 기록 리스트
```

### UX 포인트

- 꿈을 기록한 날만 강조하지 않고, “꿈이 기억나지 않은 날”도 정상적인 기록으로 보여준다.
- 날짜를 누르면 해당 날짜의 꿈 카드 또는 사라진 꿈의 발자국 카드로 이동한다.
- 월별로 자주 등장한 상징, 감정, 고양이 코멘트를 요약할 수 있다.

## 13.10 기억나지 않아요 / 아침 기분 기록 화면

### 목적

사용자가 꿈을 기억하지 못한 날에도 앱을 사용할 수 있게 한다.  
꿈 서비스의 리텐션을 위해 반드시 필요한 보완 플로우다.

### 구성

```text
“오늘 꿈은 기억나지 않아요” 상태
아침 기분 선택
떠오르는 색 선택
몸 상태 선택
오늘 가장 먼저 든 생각 입력
사라진 꿈의 발자국 카드 생성
```

### 예시 메시지

```text
오늘 꿈은 발자국만 남기고 사라졌다냥.
그래도 아침의 기분은 기록해둘 수 있다냥.
```

### MVP 적용

이 플로우는 1차 MVP에 포함한다.  
꿈을 매일 기억하지 못하는 사용자가 많기 때문에, 이 기능을 뒤로 미루면 초기 재방문 루프 검증이 약해질 수 있다.

## 13.11 자기 전 꿈 씨앗 화면

### 목적

아침 기록 중심의 서비스를 밤 루틴까지 확장한다.  
사용자가 잠들기 전 “오늘 밤 꿈에게 묻고 싶은 것”을 남기고, 다음날 다시 앱을 열도록 유도한다.

### 구성

```text
오늘 밤 꿈에게 묻고 싶은 질문 선택
짧은 메모 입력
꿈 씨앗 심기 버튼
잠들기 전 고양이 메시지
다음날 아침 꿈 기록 유도
```

### 꿈 씨앗 예시

```text
오늘 밤 꿈에게 묻고 싶은 게 있나요?

- 지금 내 마음이 궁금해
- 프로젝트에 대한 힌트가 필요해
- 그냥 이상한 꿈을 보고 싶어
- 아무것도 묻지 않고 잠들래
```

### MVP 우선순위

자기 전 꿈 씨앗은 좋은 리텐션 장치이지만, 1차 MVP에서는 범위가 넓어질 수 있다.  
따라서 2차 MVP 기능으로 분류한다.

## 13.12 설정 / 마이페이지 화면

### 목적

사용자의 기록, 해몽사, 앱 설정, 유료 기능 진입점을 관리한다.  
초기에는 단순 설정 화면으로 시작하고, 이후 Moon Pass와 고양이방 기능을 연결한다.

### 구성

```text
프로필 영역
내 꿈 기록 요약
내가 만난 고양이 해몽사
꿈 카드 저장 개수
자주 등장한 상징 요약
알림 설정
프라이버시/잠금 설정
Moon Pass 진입점
서비스 고지 및 도움말
```

### MVP 적용

1차 MVP에서는 설정 화면을 최소화한다.  
필수로 필요한 것은 서비스 고지, 기록 관리, 로그아웃 정도이며, 고양이방과 Moon Pass는 2차 이후로 확장한다.

## 13.13 하단 내비게이션

### 추천 구조

MVP 하단 탭은 3개로 시작한다.

```text
홈
백과
기록
```

확장 버전에서는 다음과 같이 구성할 수 있다.

```text
홈

기록

```

### 탭별 역할

| 탭 | 역할 | MVP 포함 여부 |
|---|---|---|
| 홈 | 꿈 입력, 기억 안 남, 오늘의 흐름 진입 | 포함 |
| 백과 | 꿈 상징 검색 및 상세 해석 열람 | 포함 권장 |
| 기록 | 꿈 달력, 꿈 카드, 사라진 꿈 기록 확인 | 포함 |
| 고양이방 | 해몽사 선택, 캐릭터 소개, 유료 캐릭터 확장 | 2차 MVP |
| 마이 | 설정, 계정, 구독, 프라이버시 | 최소 버전만 포함 |

예시 공간 요소:

```text
[수정구슬] 꿈 입력하기
[책장] 꿈 해몽 백과사전
[카드함] 내가 모은 꿈 카드
[창문] 오늘의 달빛 메시지
[고양이 방석] 해몽사 선택
```

## 14.4 캐릭터 방향

고양이는 너무 유치하면 안 된다.  
귀엽지만 신비롭고, 조용하고, 조금 엉뚱한 톤이 좋다.

추천 방향:

- 픽셀 고양이
- 검은 고양이 + 달빛 눈
- 작은 망토나 별 목걸이
- 책장 위에 앉아 있는 해몽사
- 말풍선은 짧고 부드럽게

## 14.5 핵심 비주얼 에셋

### 1. 고양이 해몽사 일러스트

용도:

- 홈 화면
- 꿈 입력 화면
- 결과 화면
- 로딩 화면
- 빈 상태 화면

방향:

- 신비로운 꿈 해몽사 분위기
- 검은 고양이 또는 달빛 눈을 가진 고양이
- 수정구슬, 별, 달빛, 연기 오브젝트와 조합
- 너무 유치하지 않고 조용한 마법사 느낌

### 2. 수정구슬 Orb 에셋

용도:

- 꿈 입력 CTA
- 분석 중 애니메이션
- 결과 생성 로딩
- 상징 검색 아이콘

방향:

- 별빛이 들어간 투명한 구슬
- 보라색, 남색, 은빛 계열
- 배경 없는 독립 에셋으로 사용 가능해야 함

### 3. 고양이 말풍선 에셋

용도:

- 고양이 해몽사의 짧은 안내 문구
- 꿈 입력 안내
- 결과 해석 코멘트
- 빈 상태 메시지

방향:

- 어두운 UI 위에서 잘 보이는 밝은 말풍선
- 부드러운 곡선
- 꿈, 달, 별 장식은 최소한으로 사용

### 4. 꿈 영수증 에셋

용도:

- 꿈 해석 결과 요약
- 공유 이미지
- 꿈 카드의 변형 포맷

방향:

- `dream receipt` 타이틀
- 오래된 종이 질감
- 검정 또는 짙은 남색 배경과 어울리는 밝은 영수증
- 고양이 캐릭터 없이 텍스트 결과를 담을 수 있는 빈 템플릿

### 5. 로딩/상태 에셋

용도:

- 꿈 분석 중
- 백과사전 검색 중
- 꿈 카드 생성 중
- 오류 또는 빈 상태

방향:

- 꿈 조각
- 작은 별가루
- 책장을 넘기는 고양이
- 수정구슬 속에서 떠오르는 상징

## 15. MVP 범위

## 15.1 MVP 목표

가장 작은 버전에서 검증할 것은 다음이다.

1. 사용자가 꿈을 입력할 의향이 있는가?
2. AI 해몽 결과가 흥미로운가?

4. 백과사전 기반 해석이 신뢰감을 주는가?
5. 다시 접속할 이유가 생기는가?

## 15.2 MVP 필수 기능

### 1차 MVP

- 홈 화면

- 꿈 카드 텍스트 생성
- 꿈 영수증 템플릿 생성
- 결과 저장
- 꿈 달력 또는 꿈 목록 조회
- 기본 백과사전 조회

### 1차 MVP에서 제외할 것

- 이미지 생성 모델 기반 카드 생성
- 복잡한 카드 디자인 커스터마이징
- 커뮤니티
- 유료 결제

- 자기 전 꿈 씨앗

### MVP 판단 기준

1차 MVP는 “예쁜 기능이 많은 앱”이 아니라, 다음 흐름이 끊기지 않는 앱이어야 한다.

```text
홈 진입
↓

↓
해석 또는 아침 기록 생성
↓
꿈 카드/꿈 영수증 확인
↓
기록 화면에 저장
```

## 15.3 MVP 성공 기준

정량 지표:

- 첫 방문 후 꿈 입력 완료율

- 다음날 재방문율
- 꿈 카드 공유 클릭률
- 평균 꿈 입력 길이

- 해몽 결과가 그럴듯한가?
- 카드가 소장하고 싶은가?
- 고양이 톤이 부담스럽지 않은가?
- 운세 앱처럼 흔하게 느껴지지 않는가?

- “단정하기보다는, 오늘은 ~를 살펴보면 좋겠습니다.”

```text
마냥 꿈해몽의 해석은 오락과 자기 성찰을 위한 감성 리딩입니다.
```

## 19. 포지셔닝 문장

### 짧은 버전

> 사라지는 꿈을 고양이가 읽고 카드로 남겨주는 감성 꿈 리딩 서비스.

### 제품 소개 버전

> 마냥 꿈해몽은 사용자의 꿈을 고양이 해몽 백과사전과 연결해 해석하고, 그 결과를 꿈 카드와 아카이브로 저장해주는 AI 기반 감성 서비스입니다.

### 차별화 강조 버전

> 단순한 AI 꿈해몽기가 아니라, 꿈속 상징을 백과사전에서 찾아 읽고 카드로 기록하는 고양이 해몽관입니다.

### 유료화 강조 버전

> 꿈이 쌓일수록 나만의 꿈 카드함과 상징 히스토리가 자라는 개인 꿈 아카이브입니다.

## 21. 최종 요약

마냥 꿈해몽은 “AI가 꿈을 해몽해주는 앱”으로 만들면 흔해질 수 있다.  
하지만 다음 구조로 잡으면 충분히 차별화할 수 있다.

```text
고양이 해몽사
+ 꿈 해몽 백과사전
+ 꿈 카드 시스템
+ 꿈 아카이브
+ 상징 히스토리
+ 오늘의 작은 처방
```

이 프로젝트의 핵심 가치는 다음이다.

> 사용자의 꿈이 그냥 사라지지 않고, 고양이 해몽관 안에서 상징과 카드로 쌓인다.

MVP에서는 개인화보다 **백과사전 기반 해석의 신뢰감과 꿈 카드의 소장성**을 먼저 검증한다.  
이후 상징 히스토리, 주간 리포트, 고급 카드 테마, 고양이 해몽사 확장을 통해 리텐션과 유료화를 설계한다.

- 오류 / 분석 실패 화면
- 백과사전 검색 결과 없음 화면

### MVP 화면 우선순위

```text
1순위: 홈, 꿈 입력, 결과, 기록/달력
2순위: 꿈 기억 안 남, 상징 백과 전체/상세
3순위: 설정/마이페이지, 자기 전 꿈 씨앗
```

## 24. 에셋 제작 현황

### 제작 방향이 잡힌 에셋

- 고양이 해몽사 일러스트
- 수정구슬 orb
- 고양이 말풍선
- dream receipt 빈 템플릿
- UI 제거 버전의 고양이/오브젝트 일러스트

### 추가로 필요한 에셋

- 로딩용 꿈 조각 아이콘
- 상징 카테고리 아이콘
- 꿈 카드 프레임
- 사라진 꿈의 발자국 카드 프레임
- 기록 없음 상태 고양이 일러스트
- 백과사전 책장 아이콘
- 분석 실패 또는 오류 상태 고양이 일러스트

### 에셋 제작 원칙

- 배경이 없는 독립 에셋과 화면 합성용 에셋을 구분해서 제작한다.
- 캐릭터는 너무 유치하지 않게, 신비롭고 조용한 고양이 해몽사 톤을 유지한다.
- UI 전체 이미지는 개발 참고용이고, 실제 구현에는 분리 가능한 에셋이 필요하다.
- 애니메이션을 고려해 고양이, 수정구슬, 연기, 촛불, 반짝임은 가능한 한 레이어를 분리한다.
