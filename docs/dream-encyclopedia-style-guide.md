# 꿈 백과사전 한국어 작성 스타일 가이드

작성일: 2026-06-02 (개정: 2026-06-03)

이 문서는 Manyang 꿈 상징 백과사전(`backend/src/data/symbol-encyclopedia.ts`)의 **한국어 내용을 어떻게 쓸지** 정한 기준이다. 데이터 구조·확장 방법은 `docs/dream-encyclopedia-guide.md`, 조사 근거는 `docs/dream-tarot-content-research.md`를 참고한다.

## 0-A. 가장 중요한 설계 원칙 — ko와 en은 서로의 번역본이 아니다

locale은 단순 언어 전환이 아니라 **문화권 전환**이다. 이 설계 의도는: 사용자의 언어가 한국어면 **동양 전통 꿈해몽**을, 영어면 **서양 전통 꿈해석**을 준다.

- **`locales.ko` = 동양 전통.** 재물·태몽·길몽/흉몽 점사, 구렁이·용·조상처럼 동양 고유 상징. ("전통적으로 ~ 길몽으로 풀려왔다")
- **`locales.en` = 서양 전통.** 융 심리학(무의식·그림자·변환), 그리스/성경 신화, 서양 꿈사전·민속의 의미. **한국 전통("In Korean tradition…")을 영어로 옮기지 않는다.**

따라서 en은 ko를 번역하지 않는다. 같은 상징이라도 문화권 해석이 다르면 **다른 내용**을 쓴다.

| 상징 | ko (동양) | en (서양) |
|---|---|---|
| 똥 | 재물이 들어오는 길몽 | 수치·해소·놓아줌 (재물 관념 없음) |
| 돼지 | 재물·복 길몽 | 풍요·탐욕·본능 |
| 뱀 | 재물·태몽 | 변환·치유·유혹(융), 무의식 |
| 용·조상 | 동양 고유 길상 | 서양 등가 없음 → 보편 심리로 |

**fortune 필드 처리:** 동양은 길흉 점사가 핵심이라 ko는 fortune을 적극 채운다. 서양은 점사 전통이 약하므로 en의 fortune은 **서양 상징 전통이 실제로 있는 경우에만** "In Western dream symbolism / Jung saw…" 형식으로 쓰고, 없으면 비운다(선택 필드). 억지로 길흉을 만들지 않는다.

매칭은 `locales[locale]`로 분기하므로 런타임상 두 문화권 내용이 공존해도 문제없다. (sceneModifier 키는 문화권마다 달라도 됨 — locale별 독립)

## 0-B. 백과사전의 역할은 "정보 전달"이다

백과사전은 RAG 해몽의 **근거 지식**이다. 화자의 말투·따뜻함은 **프롬프트에서 입힌다.** 따라서 백과사전 문장은 분위기를 잡으려 하지 말고, LLM이 끌어 쓸 **구체적인 해석 정보를 최대한 많이, 또렷하게** 담아야 한다.

기존 한국어의 문제는 두 가지였다.

1. **정보가 없다.** "영역을 알아차리는 흐름" 같은 추상 형용사는 무슨 뜻인지 알려주지 않는다(동어반복).
2. **둘러대는 동사.** "~을 비춘다 / 보여준다"는 뜻을 직접 말하지 않고 한 겹 감싼다.

→ 분위기 형용사를 버리고, **"어떤 꿈은 ~라는 뜻이다"** 형태의 직접 정보로 다시 쓴다.

---

## 1. 확정된 서술 어투 — 사전식 단언 (B안)

> **"X 꿈은 ~라는 뜻이다 / ~로 푼다."**

- 핵심 해석을 또렷하게 단언한다. "비춘다 / 보여준다 / 담겨 있다" 같은 우회 동사를 쓰지 않는다.
- 안전장치(헤지)는 개별 문장이 아니라 `safeReading`과 `avoidExpressions`가 전담한다. readings·sceneModifier까지 매번 "~수 있다"로 흐리지 않는다.
- 운명 단정(재물·태몽·진단·예언)은 여전히 금지 — `avoidExpressions`로 차단하고, 전통 길흉은 §4의 인용 형식으로만 넣는다.

---

## 2. 정보 밀도 원칙

1. **구체 > 추상.** 범주어(흐름·장면·신호·영역·기반)를 문장 끝에 붙여 때우지 않는다. 어떤 상태/행동이 어떤 뜻인지 말한다.
2. **변형(variation)을 많이 담는다.** 한 상징의 가치는 "꿈의 어떤 변형 → 어떤 뜻"을 얼마나 촘촘히 커버하느냐에 있다. 이 정보는 주로 `sceneModifiers`가 진다.
3. **나쁜 의미도 또렷이.** `shadowReadings`·`fortune.cautious`에서 어려움을 직접 말하되, 예언이 아니라 점검·환기로 닫는다.
4. **en 로케일 동기화.** 한국어를 정보형으로 다시 쓰면 영어도 같은 정보를 담도록 맞춘다.

---

## 3. 번역체 / 우회 표현 블랙리스트

| 금지 | 왜 | 대체 |
|---|---|---|
| "~을 **비춘다 / 보여준다**" | 뜻을 직접 안 말하고 둘러댐 | "~라는 뜻이다 / ~로 푼다" |
| "~을 알아차리는 **흐름**" | flow 직역, 정보 없음 | "~한 꿈은 ~라는 뜻이다" |
| "~을 확인하는 **장면**" | scene 직역 | 동사로 직접 끝내기 |
| "여러 **신호**가 움직이는" | signal이 추상 | 구체 동작("여러 일이 한꺼번에 벌어지는") |
| 추상어 + 시적 이미지("기반에 모인 달빛") | 비논리적 결합 | 물상끼리("내 자리에 고인 달빛") |

원칙: **흐름 / 장면 / 신호 / 영역 / 기반을 문장 끝 접미사로 쓰지 않는다.** (단 `sceneModifiers`의 "~한 꿈 — ~" 묘사부는 예외)

## 4. 필드별 작성 지침

**키워드 필드 (검색·매칭용, 자연스러움보다 적중 우선)**
- `aliases` / `searchText`: 사용자가 실제로 칠 표기를 넓게.
- `coreMeanings`: 짧은 명사구 태그.

**정보 필드 (B안 직접 서술 적용)**
- `lightReadings` (2개): 긍정 쪽 핵심 해석. "~한 꿈은 ~라는 뜻이다."
- `shadowReadings` (2개): 부담 쪽 핵심 해석. 호명하되 예언 금지.
- `sceneModifiers` (**항목당 5~7개로 확장**): 정보의 핵심. 각 항목은 **"…한 꿈 — …라는 뜻이다(+길흉 valence)"** 형식. `triggerTerms`로 RAG 매칭도 되게 실제 검색어를 넣는다. 상태(넓다/갈라지다)·행동(잡다/물리다)·색·수량 등 변형을 촘촘히.
- `fortune.auspicious` / `cautious`: 전통 길흉을 **적극 인용**. 단 "전통적으로 ~로 풀려왔다 / 보았다"처럼 거리를 둔 인용 형식. cautious는 점검·환기로 닫는다. 단정 예언("재물이 들어온다")은 금지.

**창작 시드 (정보 아님, 생성 보조)**
- `metaphorHooks` / `cardTitleSeeds`: 물상 이미지 중심, 추상명사 단독 금지.
- `smallPrescriptions`: 짧은 권유.
- `safeReading`: 안전한 기본 해석 한 줄. 여기서 "~로 읽을 수 있어요" 헤지는 의도된 장치라 유지.

## 5. 확정 예시: "우리 땅" (B안 적용)

```js
coreMeanings:   ["삶의 기반", "생활 터전", "소유와 책임", "가족·생활권"],
lightReadings: [
  "내가 가꾼 땅이 넓고 단단해 보이는 꿈은 삶의 토대가 안정돼 있다는 뜻으로 푼다",
  "낯익은 내 땅을 다시 밟는 꿈은 흔들리던 마음이 제자리를 찾았다는 뜻이다",
],
shadowReadings: [
  "내 땅을 누가 침범하거나 빼앗으려는 꿈은 내 영역을 지켜야 한다는 압박과 경계심이 담긴 꿈이다",
  "땅이 갈라지거나 꺼지는 꿈은 발 딛은 생활 기반이 흔들린다는 뜻이다",
],
sceneModifiers: {
  wideOrVast:        "땅이 끝없이 넓게 펼쳐진 꿈 — 기반과 살림의 규모가 커진다는 전통적 길몽이다",
  buyingOrExpanding: "땅을 새로 사거나 넓히는 꿈 — 생활 기반과 재물이 넓어진다고 풀어온 길몽이다",
  fertileGround:     "흙이 비옥하거나 작물이 자라는 꿈 — 아직 드러나지 않은 가능성이 기반에서 자란다는 뜻이다",
  filledWithAnimals: "내 땅에 짐승이 가득한 꿈 — 여러 일이 한꺼번에 생활권에서 벌어진다는 뜻이다",
  unfamiliarOnLand:  "익숙한 내 땅에 낯선 것이 들어온 꿈 — 내 영역의 경계가 시험받고 있다는 뜻이다",
  crackedOrSinking:  "땅이 갈라지거나 꺼지는 꿈 — 생활 기반을 한 번 점검하라는 경고로 푼다",
},
// fortune: 전통 길흉을 인용 형식으로 적극 포함 (auspicious: 넓은 땅·매입 / cautious: 갈라짐·빼앗김)
```

핵심: "비춘다"류 우회 동사 제거 → "~라는 뜻이다 / ~로 푼다" 직접 서술. sceneModifier를 3 → 6개로 늘려 변형 정보를 촘촘히. 전통 길흉은 fortune에 인용으로.

## 6. 작성 후 자가 점검

- [ ] "비춘다 / 보여준다 / 담겨 있다" 같은 우회 동사를 안 썼는가? ("~라는 뜻이다"로 직접 말했는가?)
- [ ] 문장 끝이 "흐름 / 장면 / 신호"로 때워지지 않았는가?
- [ ] `sceneModifiers`가 5개 이상이고, 각각 "어떤 꿈 → 어떤 뜻"을 담는가?
- [ ] 전통 길흉을 fortune에 인용 형식으로 적극 넣었는가? (단정 예언은 아닌가?)
- [ ] `avoidExpressions`의 단정(재물·태몽·진단·예언)을 본문에서 쓰지 않았는가?
- [ ] `en` 로케일과 정보가 일치하는가?

## 7. 제거된 필드 (2026-06-03)

`interpretationLenses` · `cultureNotes` · `tensionAxis`는 런타임 미사용이었고, locale=문화 결정으로 lens는 개념적으로 중복돼 **계약·데이터·테스트에서 완전히 제거**했다. 유지: `embeddingProfile`(RAG 청크 계약), `cardTitleSeeds`(카드 기능 미래 연결 여지). 분류 필드(facets/symbolRole/safetyLevel/accessTier/relatedIds/sourceBasis/universalMeanings)는 실사용이라 유지.
