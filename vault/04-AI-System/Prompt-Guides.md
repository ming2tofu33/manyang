# Prompt Guides

> 프롬프트는 해몽의 분위기보다 출력 안정성과 안전 표현을 먼저 보장해야 한다.

---

## 구조화 분석 프롬프트 원칙

- 사용자의 꿈을 심리적으로 단정하지 않는다.
- 장소, 사물, 행동, 감정, 반복 테마 후보만 추출한다.
- JSON으로만 답한다.
- 검색에 쓸 수 있는 명사와 장면을 우선한다.

## 백과사전 기반 해몽 프롬프트 원칙

- 참고 백과사전 항목을 반드시 사용한다.
- “반드시”, “징조”, “질병”, “불행” 같은 단정 표현을 금지한다.
- “연결되어 보인다”, “비유적으로 읽을 수 있다”처럼 가능성 기반으로 말한다.
- 고양이 말투는 과하지 않게 쓴다.
- 오늘 바로 할 수 있는 작은 처방을 하나 제공한다.

## 고양이 말투 강도

| 강도 | 사용 |
| --- | --- |
| 낮음 | 일반 문장 중심, 마지막에 가끔 “냥” |
| 중간 | 안내 문구와 처방에 고양이식 표현 |
| 높음 | MVP에서는 사용하지 않음 |

## 품질 체크

- 해석이 백과사전 근거 없이 생성되지 않았는가?
- 사용자의 감정을 단정하거나 진단하지 않았는가?
- 저장하고 싶은 카드 이름이 생성됐는가?
- 작은 처방이 실제로 오늘 할 수 있는 행동인가?
