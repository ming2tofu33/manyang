# Character Reading Retention and Monetization

> 마냥은 타로를 주 소비 콘텐츠로, 꿈과 감정 기록을 개인화 자산으로, 고양이 관계와 수집을 재방문 동기로 사용하는 감성 운세·리딩 서비스로 확장한다.

---

## 결정 요약

- 제품의 확장 포지션은 **고양이가 읽어주는 감성 운세·리딩 서비스**다.
- 타로는 가장 자주 소비되는 전면 콘텐츠다.
- 꿈 해몽은 마냥만의 차별화 콘텐츠와 개인화 기억으로 사용한다.
- 캐릭터 관계가 방문 이유를, 개인 리딩 서재가 장기 가치와 과금 이유를 담당한다.
- 수익 모델은 무료 데일리 콘텐츠, 달조각 소비 콘텐츠, Moon Pass를 결합한다.
- 웹은 무료 체험·검색 유입·앱 설치 전환을 담당하고, 앱은 관계·수집·결제의 본체가 된다.

## 제품 가치 구조

```text
캐릭터를 보러 방문
→ 무료 오늘의 타로
→ 리딩 반응 또는 기록
→ 고양이가 사용자를 기억
→ 관계·대사·장면·수집이 확장
→ 고민별 달조각 리딩
→ 주간 편지와 월간 정산
```

| 요소 | 역할 |
| --- | --- |
| 고양이 IP | 방문, 애착, 공유 |
| 타로 | 반복 소비와 첫 결제 |
| 꿈·감정 기록 | 마냥만의 개인화 데이터 |
| 개인 리딩 서재 | 장기 보유 가치 |
| 달조각 | 고민별 리딩과 꾸미기 소비 |
| Moon Pass | 기억 기반 리딩과 관계 확장 |

## 캐릭터 원칙

고양이는 시각 테마에 머물지 않고 동반자 역할을 맡는다. 다만 리딩의 정확도나 안전 기준을 고양이별로 다르게 만들지는 않는다.

- 카드 의미, 꿈 상징 근거, 안전 정책은 공통 리딩 엔진이 담당한다.
- 인사, 질문 방식, 반응, 편지, 관계 에피소드, 표정과 장면은 고양이마다 다르다.
- 특정 고양이가 더 정확하거나 미래를 더 잘 맞힌다고 표현하지 않는다.
- 관계 단계는 돈으로 구매할 수 없다.
- 사용하지 않은 기간이 있어도 관계는 하락하지 않는다.
- 확률형 뽑기와 불안 자극형 결제는 사용하지 않는다.

## 주 동반자와 다른 고양이

- 사용자는 한 마리를 주 동반자로 선택한다.
- 주 동반자와 가장 깊은 관계·편지·기억 연결을 경험한다.
- 다른 고양이도 각각의 에피소드와 수집 항목을 가진다.
- 주 동반자를 바꿔도 기존 고양이의 관계 진행은 보존한다.
- 선택된 고양이만 그날의 관계 성장을 받는다.

## 첫 버전 관계 3단계

| 단계 | 조건 | 해금 |
| --- | --- | --- |
| 첫 인사 | 주 동반자 선택 | 기본 인사·표정, 오늘의 타로 |
| 익숙한 발자국 | 서로 다른 3일에 의미 있는 활동 | 첫 편지, 새 표정, 고양이별 에피소드, 기본 소품 1개 |
| 마음을 아는 사이 | 서로 다른 7일 활동과 리딩 반응 1회 | 주간 편지, 특별 장면, 전용 스탬프, 개인화 리딩 미리보기 |

의미 있는 활동은 하루에 최대 한 번만 관계 성장으로 인정한다.

- 오늘의 타로 완료
- 고민별 리딩 완료
- 리딩에 감정 또는 한 줄 반응 남기기
- 꿈·아침·밤 기록 남기기
- 고양이 편지에 답하기

결제를 반복해도 같은 날 관계가 더 빨리 성장하지 않는다.

## 첫 확장 버전 범위

캐릭터와 수익화를 함께 검증하는 균형형 범위를 사용한다.

- 주 동반자 선택
- 관계 3단계
- 무료 오늘의 한 장
- 기존 질문형 한 장
- 고민별 3장 리딩 3종
- 고양이의 주간 편지 1종
- 고양이별 짧은 관계 에피소드
- 해금 가능한 기본 소품 약 6개
- Moon Pass 개인화 리딩 미리보기
- 기존 꿈·감정 기록 연결

첫 버전에서 제외한다.

- 자유 배치형 방 편집기
- 대량의 의상과 가구
- 신규 점술 카테고리
- 복잡한 다중 재화
- 확률형 수집
- 무제한 유료 리딩

## 콘텐츠와 과금 경계

### 무료

- 오늘의 한 장 하루 1회
- 로그인 사용자의 질문형 한 장 하루 1회
- 기본 꿈 해몽과 기록
- 기본 관계 에피소드와 관계 3단계
- 리딩 반응 남기기

리딩 반응은 짧은 선택형으로 시작한다.

- 마음에 와닿았어
- 조금 더 생각해볼래
- 아직 잘 모르겠어

### 달조각 소비 콘텐츠

| 콘텐츠 | 카드 구성 | 안전 기준 |
| --- | --- | --- |
| 관계의 온도 | 내 마음·관계의 흐름·내가 할 수 있는 것 | 상대의 속마음을 단정하지 않음 |
| 선택의 갈림길 | 선택 A의 흐름·선택 B의 흐름·판단 기준 | 결정을 대신하지 않음 |
| 현실의 실마리 | 막힌 지점·사용 가능한 자원·다음 행동 | 투자·재정 결과를 보장하지 않음 |

추가 달조각 사용처:

- 꿈+타로 교차 리딩
- 특별 캐릭터 에피소드
- 의상과 방 테마
- 카드 프레임과 영수증 장식
- 시즌 콘텐츠

### Moon Pass

- 정기 달조각 또는 달조각 혜택
- 과거 타로·꿈·감정 기록을 참고하는 개인화
- 주 동반자의 주간 편지
- 월간 리딩 정산서
- 꿈+타로 교차 리딩 혜택
- 확장 캐릭터 에피소드
- 프리미엄 의상·방·영수증 테마
- 완성된 유료 리딩의 지속 보관

정확한 가격, 지급 달조각 수, 콘텐츠별 소모량은 아직 확정하지 않는다.

## 재화 원칙

### 발자국

- 활동과 기록으로 얻는다.
- 관계·기록의 흔적이며 현금으로 판매하지 않는다.
- 기본 소품이나 장면 해금에 사용할 수 있다.

### 달조각

- 앱에서만 구매하고 사용한다.
- Apple In-App Purchase와 Google Play Billing을 사용한다.
- 구매한 달조각은 만료되지 않는다.
- 구매분과 프로모션 지급분은 서버 장부에서 구분한다.
- 관계 단계나 애정을 구매하는 데 사용할 수 없다.
- 리딩 생성 실패 시 차감하지 않거나 즉시 복구한다.
- 가격과 원화 환산 정보를 숨기지 않는다.

## 웹과 앱의 역할

| 웹 | 앱 |
| --- | --- |
| 검색·SNS 유입 | 주 동반자와 관계 성장 |
| 오늘의 한 장 무료 체험 | 질문형·고민별 리딩 |
| 기본 꿈 해몽 | 달조각 구매·사용 |
| 고양이 소개와 선택 | 고양이 방·편지·에피소드 |
| 상징 백과 | 의상·가구·수집 |
| 완성된 리딩 결과 열람 | Moon Pass와 개인화 리딩 |
| 앱 설치 유도 | 알림과 지속 방문 |

웹은 결과를 보여주기 전에 앱 설치를 강요하지 않는다.

```text
웹 무료 리딩 완료
→ 이 카드를 고양이가 기억하게 할지 제안
→ 앱 설치 또는 앱 열기
→ 동일 계정으로 기록 불러오기
→ 첫 관계 에피소드
```

앱 출시 전 웹에서는 결제가 가능한 것처럼 오해시키지 않고 다음 퍼널을 측정한다.

```text
프리미엄 리딩 노출
→ 앱에서 열기 클릭
→ 출시 알림 신청
→ 앱 설치
→ 기존 계정 로그인
→ 첫 달조각 구매
```

## 결제 및 실패 처리

- 달조각 구매와 사용은 앱에서만 가능하다.
- 앱에서 완성한 유료 리딩은 웹에서도 열람할 수 있다.
- 구매 영수증은 서버에서 검증한 뒤 달조각을 지급한다.
- 외부 거래 ID는 한 번만 처리해 중복 지급을 막는다.
- 같은 질문의 반복 소비 전에는 기존 결과를 먼저 안내한다.
- 결제 또는 생성 실패 시 동일 카드와 질문으로 재시도할 수 있다.
- 이미 완성한 리딩과 관계 단계는 Moon Pass 종료 후에도 유지한다.
