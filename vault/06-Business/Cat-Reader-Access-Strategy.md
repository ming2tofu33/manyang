# Cat Theme Monetization Strategy

> 고양이는 해몽 결과를 바꾸는 해몽사가 아니라, 동일한 꿈해몽 엔진 위에 얹는 동행자 테마와 스킨이다.

> [!IMPORTANT]
> 이 문서의 **고양이를 시각 테마로만 제한하는 장기 방향은 2026-07-11 결정으로 대체되었다.** 리딩 정확도와 안전 기준을 공통으로 유지하는 원칙은 계속 유효하며, 관계·편지·에피소드·수집 확장은 [[Character-Reading-Retention-and-Monetization]]을 따른다.

---

## 결정 요약

마냥의 고양이는 더 이상 **해몽 관점, 감정 톤, 결과 문체, 해석 정확도**를 나누는 장치로 쓰지 않는다.

어떤 고양이를 선택해도 꿈 해몽의 핵심 결과는 같아야 한다.

- 꿈 요약
- 주요 상징
- 감정 흐름
- 해몽 본문
- 오늘의 작은 처방
- 안전 문구
- 영수증 문체

고양이 선택으로 달라질 수 있는 것은 시각적 경험뿐이다.

- 홈 화면 고양이 이미지
- 로딩 화면 고양이/배경
- 결과 영수증 도장, 장식, 테마 색
- 프로필 방/배경
- 저장 이미지/PDF 템플릿
- 시즌 의상, 스킨, 스탬프

이 방향이 더 나은 이유는 사용자가 "어떤 고양이에게 받느냐에 따라 결과가 달라진다"고 느끼면 해몽 신뢰도가 떨어질 수 있기 때문이다. 마냥의 신뢰는 **고양이별 다른 말**이 아니라 **일관된 해몽 기준, 백과사전 근거, 안전한 표현, 저장하고 싶은 결과물**에서 만들어야 한다.

## 제품 원칙

- 고양이는 해몽사가 아니라 동행자/테마/스킨이다.
- 고양이 선택은 해몽 결과의 정확도, 깊이, 말투를 바꾸지 않는다.
- 유료화는 "더 맞는 해몽"이 아니라 "더 예쁘게 보관하고 꾸미는 경험"에 둔다.
- 잿빛냥/회색냥은 프리미엄 테마가 될 수 있지만, 더 좋은 해몽을 주는 캐릭터로 표현하지 않는다.
- 타로는 고양이별 해몽 차이가 아니라 별도의 프리미엄 추가 리딩으로 둔다.
- 기본 꿈 기록, 기본 꿈 해몽, 기본 꿈 영수증은 무료 경험으로 유지한다.

## 테마별 역할

| 테마 | 접근 정책 | 역할 | 비즈니스 목적 |
| --- | --- | --- | --- |
| 검은냥 | 무료 기본 | 기본 밤하늘 테마 | 첫 경험의 기준선 |
| 하얀냥 | 무료 또는 초기 해금 | 부드러운 달빛 테마 | 취향 선택과 정서적 안정감 |
| 치즈냥 | 무료 또는 초기 해금 | 따뜻한 노을 테마 | 밝은 분위기 선택지 |
| 잿빛냥/회색냥 | Moon Pass 또는 시즌 프리미엄 | 깊은 서재/타로 무드 테마 | 프리미엄 소장 가치 |

테마 설명은 해석 관점이 아니라 시각적 분위기로 쓴다.

좋은 예:

```text
잿빛냥의 달빛 서재 테마로 꿈 영수증을 남겨요.
```

피해야 할 예:

```text
잿빛냥이 더 깊고 정확하게 꿈과 타로를 읽어드려요.
```

## 무료와 유료의 경계

무료에서 제공할 수 있는 것:

- 검은냥 기본 테마
- 하얀냥/치즈냥 기본 테마 또는 온보딩 해금
- 기본 꿈 영수증
- 기본 저장/공유 이미지
- 기본 홈/로딩 테마

유료 후보:

- 잿빛냥 프리미엄 테마
- 고급 꿈 영수증 템플릿
- 고양이 의상/방/스탬프
- 시즌 한정 테마팩
- 고해상도 이미지/PDF 내보내기
- 월간 꿈 정산서 전용 테마
- 꿈 책/PDF 테마

타로는 별도 프리미엄 기능으로 다룬다.

- 타로는 "미래 예언"이 아니라 "꿈을 다른 상징 카드로 한 번 더 비춰보는 리딩"이다.
- 타로 리딩을 제공하더라도 꿈 해몽 본문이 고양이 때문에 달라져서는 안 된다.
- UI에서는 "테마 선택"과 "타로 추가 리딩"을 분리한다.

## 결과 화면 기준

해몽 결과의 구조는 모든 고양이에서 동일하다.

| 영역 | 고양이 영향 여부 | 설명 |
| --- | --- | --- |
| 꿈 요약 | 영향 없음 | 동일 입력이면 동일 기준으로 요약 |
| 상징 해석 | 영향 없음 | RAG/백과사전 근거 기준 |
| 감정 흐름 | 영향 없음 | 안전한 reflection 문장 유지 |
| 작은 처방 | 영향 없음 | 고양이별 행동 처방 차이 금지 |
| readerNote | 제거 또는 공통화 | 특정 고양이 말투/관점으로 분기하지 않음 |
| 영수증 장식 | 영향 가능 | 도장, 색, 배경, 일러스트만 변경 |
| 저장 이미지/PDF | 영향 가능 | 템플릿, 장식, 해상도, 시즌 스킨 |

결과 화면 카피:

```text
선택한 고양이 테마로 꿈 영수증을 꾸몄어요.
```

프리미엄 테마 CTA:

```text
잿빛냥 달빛 서재 테마로 저장해볼까요?
```

타로 CTA:

```text
꿈을 카드 상징으로 한 번 더 비춰보기
```

## 데이터 모델 방향

현재 코드에는 `CatReader`와 `catReaderType`이라는 이름이 남아 있다. 다음 구현에서는 의미를 바꿔야 한다.

권장 최종 모델:

```ts
type CatThemeAccess = "free" | "premium";

type CatThemeId = "black_cat" | "white_cat" | "cheese_cat" | "gray_cat";

type CatTheme = {
  id: CatThemeId;
  name: string;
  access: CatThemeAccess;
  visualRole: string;
  shortDescription: string;
  assetKey: string;
  backgroundKey: string;
  receiptThemeKey: string;
  ctaLabel: string;
  lockedLabel?: string;
};
```

전환 중에는 DB와 localStorage 호환을 위해 `catReaderType`을 바로 삭제하지 않아도 된다. 다만 의미는 "해몽사 타입"이 아니라 "선택한 고양이 테마 id"로 제한한다.

## API 원칙

꿈 분석 요청은 고양이 id로 해몽 프롬프트를 분기하지 않는다.

권장 방향:

```ts
type DreamAnalysisRequest = {
  dreamText: string;
  dreamDate: string;
  wakeMood?: string;
  locale?: "ko" | "en";
  nightContext?: NightContext;
  readingMode?: "basic" | "detailed";
};

type DreamPresentationRequest = {
  analysisId: string;
  catThemeId: CatThemeId;
};
```

MVP에서는 API를 크게 나누지 않더라도 다음 원칙은 지킨다.

- `catReaderType`을 LLM prompt의 persona로 넣지 않는다.
- `gray_cat`이라는 이유만으로 `readingKind`를 detailed로 바꾸지 않는다.
- 상세 해몽은 `readingMode` 또는 명시적 유료 권한으로만 결정한다.
- 저장 데이터에는 고양이를 "theme id"로 보관한다.

## 측정 이벤트

| 이벤트 | 트리거 | 확인할 것 |
| --- | --- | --- |
| `cat_theme_selector_viewed` | 테마 선택 영역 노출 | 사용자가 테마 선택을 보는가 |
| `cat_theme_selected` | 고양이 테마 선택 | 어떤 시각 테마가 선호되는가 |
| `premium_cat_theme_viewed` | 프리미엄 테마 잠금 노출 | 유료 테마 제안 노출 수 |
| `premium_cat_theme_interest_clicked` | 프리미엄 테마 CTA 클릭 | 스킨/테마 유료 의향 |
| `tarot_addon_interest_clicked` | 타로 추가 리딩 CTA 클릭 | 타로 add-on 수요 |
| `premium_receipt_interest_clicked` | 프리미엄 영수증 CTA 클릭 | 저장물 유료 의향 |

MVP 판단 기준:

- 테마 선택률이 높으면 캐릭터/스킨 선택의 의미가 있다.
- 프리미엄 테마 CTA가 눌리면 잿빛냥을 유료 테마로 유지한다.
- 타로 CTA는 테마 CTA와 분리해서 측정한다.
- 고양이별 재해석 클릭률은 더 이상 핵심 KPI가 아니다.

## 구현 우선순위

### MVP 정리

1. 고양이 선택 UI 문구를 "해몽사 선택"에서 "테마 선택"으로 바꾼다.
2. `catReaderType`이 해몽 결과/문체/프롬프트를 바꾸는 경로를 제거한다.
3. 모든 고양이가 동일한 해몽 엔진과 동일한 output contract를 사용하게 한다.
4. `gray_cat`이 상세 해몽 권한을 자동으로 만들지 않게 한다.
5. 잿빛냥/회색냥은 프리미엄 테마 잠금 CTA로만 남긴다.
6. 결과 영수증에는 선택한 고양이 테마의 장식만 적용한다.
7. 타로는 별도 add-on CTA와 이벤트로 분리한다.

### Post-MVP

1. `CatReader` 타입과 파일명을 `CatTheme`으로 rename한다.
2. DB 컬럼 `cat_reader_type`을 `cat_theme_id`로 migration한다.
3. 프리미엄 영수증 테마팩과 고양이 의상/방 꾸미기를 실제 상품 후보로 검증한다.
4. 타로 add-on의 안전 문구, 1장 카드 구조, 비용 제한을 별도 문서/코드로 구현한다.
5. 월간 꿈 정산서에는 선택한 테마를 장식으로만 적용한다.

## 완료 기준

- 어떤 고양이를 선택해도 동일한 꿈 입력의 해몽 본문/문체/작은 처방이 달라지지 않는다.
- 고양이 선택은 홈, 로딩, 영수증, 저장 이미지의 시각 테마에만 영향을 준다.
- 프리미엄 고양이 CTA는 "더 정확한 해몽"이 아니라 "프리미엄 테마/저장물"로 표현된다.
- 타로는 고양이 테마가 아니라 별도 프리미엄 add-on으로 측정된다.
- 문서와 코드에서 "고양이별 관점 차이"를 유료화 근거로 삼지 않는다.
