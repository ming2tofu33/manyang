# Cat Reader Access Strategy

> 무료 해몽사 3마리로 차이를 체감시키고, 회색고양이는 깊은 분석과 타로 상징 리딩을 묶은 연간 프리미엄 해몽사로 둔다.

---

## 결정 요약

고양이 해몽사는 단순 캐릭터 스킨이 아니라 **해몽 관점, 감정 톤, 결과 문체, 유료화 이유**를 만드는 제품 장치다.

MVP와 초기 유료화에서는 다음 정책을 따른다.

| 고양이 | 접근 정책 | 핵심 역할 | 비즈니스 목적 |
| --- | --- | --- | --- |
| 검은냥 | 무료 | 상징 중심 기본 해몽 | 대표 해몽사, 제품 기준선 |
| 하얀냥 | 무료 | 감정 위로형 해몽 | 불안한 꿈의 거부감 완화 |
| 치즈냥 | 무료 | 오늘의 행동/처방형 해몽 | 기록 후 다음 행동 유도 |
| 회색냥 | 연간 프리미엄 | 깊은 꿈 분석 + 타로 상징 리딩 | 고가 구독의 대표 가치 |

무료 해몽사 3마리를 먼저 제공하는 이유는 사용자가 "고양이마다 관점이 다르다"는 차이를 직접 경험해야 회색고양이의 프리미엄 가치를 이해할 수 있기 때문이다.

## 제품 원칙

- 무료 고양이 3마리는 우열이 아니라 관점 차이를 만든다.
- 회색고양이는 더 긴 해몽사가 아니라 다른 수준의 분석 경험이어야 한다.
- 타로는 미래 예언이 아니라 꿈을 다른 상징 체계로 다시 비춰보는 장치로 표현한다.
- 고양이별 해석은 결론이 충돌하지 않고 초점만 달라져야 한다.
- 기본 꿈 기록, 기본 해몽, 기본 꿈 영수증은 무료 경험으로 유지한다.

## 고양이별 구현 스펙

### 검은냥

| 항목 | 내용 |
| --- | --- |
| id | `black_cat` |
| 표시명 | 검은냥 |
| 접근 | 무료 |
| 한 줄 역할 | 꿈속 상징을 조용히 읽어주는 기본 해몽사 |
| 핵심 관점 | 상징, 장면, 백과사전 근거 |
| 감정 톤 | 신비롭고 차분함 |
| 결과 강점 | 꿈 해몽의 기준선 제공 |
| 추천 상황 | 첫 사용, 일반 꿈, 상징이 선명한 꿈 |

결과 문체:

```text
이 꿈에서 가장 먼저 읽히는 상징은 복도와 신발이에요.
복도는 전환의 구간처럼, 신발은 준비 상태의 감각처럼 읽을 수 있다냥.
```

CTA 예시:

```text
검은냥에게 꿈 비춰보기
```

### 하얀냥

| 항목 | 내용 |
| --- | --- |
| id | `white_cat` |
| 표시명 | 하얀냥 |
| 접근 | 무료 |
| 한 줄 역할 | 불안한 꿈을 부드럽게 다독이는 해몽사 |
| 핵심 관점 | 감정 안정, 위로, 자기 보호 |
| 감정 톤 | 부드럽고 따뜻함 |
| 결과 강점 | 무서운 꿈, 찝찝한 꿈의 부담 완화 |
| 추천 상황 | 불안, 쫓김, 상실, 어두운 장소, 악몽 |

결과 문체:

```text
이 꿈은 무서운 예고라기보다, 마음이 긴장을 안전하게 꺼내본 장면에 가까워 보여요.
오늘은 이 감정을 오래 붙잡기보다 따뜻한 물 한 잔처럼 천천히 내려놓아도 괜찮다냥.
```

CTA 예시:

```text
하얀냥에게 부드럽게 읽어달라고 하기
```

### 치즈냥

| 항목 | 내용 |
| --- | --- |
| id | `cheese_cat` |
| 표시명 | 치즈냥 |
| 접근 | 무료 |
| 한 줄 역할 | 꿈에서 오늘의 작은 행동을 찾아주는 해몽사 |
| 핵심 관점 | 실행, 루틴, 작은 처방 |
| 감정 톤 | 밝고 현실적임 |
| 결과 강점 | 해몽 후 바로 할 수 있는 행동 제안 |
| 추천 상황 | 해야 할 일이 많을 때, 조급함, 준비, 선택, 시작 |

결과 문체:

```text
이 꿈은 오늘 모든 걸 해결하라는 말은 아니에요.
대신 가장 작은 준비물 하나만 확인해보면 좋겠다냥. 신발을 찾는 꿈이었다면, 오늘은 이동 전에 필요한 것 하나만 챙겨보세요.
```

CTA 예시:

```text
치즈냥에게 오늘의 처방 받기
```

### 회색냥

| 항목 | 내용 |
| --- | --- |
| id | `gray_cat` |
| 표시명 | 회색냥 |
| 접근 | 연간 프리미엄 |
| 한 줄 역할 | 꿈 기록과 타로 상징을 함께 읽는 프리미엄 해몽사 |
| 핵심 관점 | 반복 패턴, 상징 히스토리, 타로 상징 리딩 |
| 감정 톤 | 조용하고 깊음 |
| 결과 강점 | 축적된 기록을 바탕으로 한 종합 리딩 |
| 추천 상황 | 반복되는 꿈, 월간 정산, 깊은 해석 니즈 |

회색냥은 단순히 더 긴 해몽을 제공하지 않는다. 다음 가치를 함께 제공해야 프리미엄으로 납득된다.

- 현재 꿈의 상징 분석
- 최근 꿈 기록과의 연결
- 반복 상징 또는 감정 흐름 언급
- 오늘의 타로 1장 상징 리딩
- 꿈 해몽과 타로 상징을 엮은 종합 코멘트
- 월간 꿈 정산서의 특별 코멘트

결과 문체:

```text
이번 꿈의 복도는 오늘만의 장면이라기보다, 최근 기록에 반복되는 '전환'의 흐름과 이어져 보여요.
오늘의 타로 상징은 은둔자입니다. 미래를 예언한다기보다, 지금은 서두르기보다 조용히 방향을 비춰보는 태도와 잘 맞아요.
```

CTA 예시:

```text
회색냥의 꿈+타로 리딩 보기
```

## 무료 경험 설계

무료 사용자는 검은냥, 하얀냥, 치즈냥을 모두 경험할 수 있다.

권장 UX:

1. 첫 해몽은 검은고양이를 기본 선택으로 둔다.
2. 결과 화면 하단에서 다른 무료 고양이 관점을 제안한다.
3. 사용자가 같은 꿈을 다른 고양이로 다시 읽을 수 있게 하되, MVP에서는 결과 전체 재생성보다 짧은 추가 코멘트로 시작한다.
4. 고양이 선택 화면에는 각 고양이의 차이를 한 문장으로 보여준다.

해몽사 선택 카피:

| 고양이 | 선택 문구 |
| --- | --- |
| 검은냥 | 꿈속 상징을 조용히 읽어요 |
| 하얀냥 | 불안한 꿈을 부드럽게 다독여요 |
| 치즈냥 | 오늘 할 수 있는 작은 행동을 찾아요 |
| 회색냥 | 꿈 기록과 타로 상징을 함께 읽어요 |

## 회색고양이 프리미엄 설계

회색냥은 단독 잠금 캐릭터가 아니라 **연간 Moon Pass의 대표 얼굴**로 사용한다.

연간 프리미엄에 묶을 가치:

- 회색냥 꿈+타로 리딩
- 월간 꿈 정산서
- 상징 히스토리 심화 보기
- 무제한 꿈 영수증 보관함
- 고급 꿈 영수증 템플릿
- 고해상도 또는 PDF 저장

회색냥 잠금 화면 카피:

```text
회색냥은 지난 꿈 기록과 오늘의 타로 상징을 함께 읽는 프리미엄 해몽사예요.
지금은 준비 중입니다. 열리면 가장 먼저 알려드릴게요.
```

유료화 이후 카피:

```text
연간 Moon Pass로 회색냥의 꿈+타로 리딩과 월간 꿈 정산서를 열어보세요.
```

## 타로 리딩 안전 원칙

타로는 마냥의 안전 원칙을 해치지 않도록 **예언이 아니라 상징 리딩**으로 제한한다.

금지:

- 곧 어떤 일이 일어난다고 단정하기
- 연애, 건강, 사고, 죽음, 질병을 예언처럼 말하기
- 사용자의 선택을 대신 결정하기
- 공포나 불안을 자극하는 카드 해석

권장:

- "미래 예언이 아니라 오늘 꿈을 다른 상징으로 비춰보는 카드입니다."
- "이 카드는 지금의 태도를 떠올려보게 합니다."
- "단정하기보다, 오늘은 이런 질문을 곁에 두면 좋겠습니다."

타로 결과 구성:

| 섹션 | 설명 |
| --- | --- |
| 오늘의 카드 | 카드 이름과 짧은 상징 |
| 꿈과의 연결 | 꿈속 주요 상징과 카드 상징의 접점 |
| 오늘의 질문 | 사용자가 자기 성찰에 쓸 질문 1개 |
| 작은 처방 | 오늘 할 수 있는 낮은 부담의 행동 1개 |

## 결과 화면 적용

기본 결과 화면은 꿈 영수증을 중심으로 둔다.

고양이별 추가 영역:

| 고양이 | 결과 화면 차이 |
| --- | --- |
| 검은냥 | 상징 근거와 백과사전 링크를 강조 |
| 하얀냥 | 감정 이름 붙이기와 안정 문장을 강조 |
| 치즈냥 | 오늘의 작은 처방과 체크 가능한 행동을 강조 |
| 회색냥 | 최근 기록 연결, 타로 카드, 월간 정산 연결을 강조 |

무료 고양이 재해석 CTA:

```text
다른 고양이 관점으로도 읽어볼까요?
```

회색냥 CTA:

```text
회색냥에게 꿈과 타로를 함께 물어보기
```

MVP에서는 회색고양이 CTA 클릭 시 실제 결제 대신 관심 이벤트를 남긴다.

## API 요청/응답 영향

꿈 분석 요청에는 선택한 고양이 id를 포함한다.

```ts
type DreamAnalysisRequest = {
  dreamText: string;
  dreamDate: string;
  wakeMood?: string;
  catReaderType: CatReaderId;
};
```

결과 응답에는 사용된 고양이와 결과 강조 타입을 포함한다.

```ts
type DreamAnalysisResponse = {
  dreamId: string;
  analysisId: string;
  reader: {
    id: CatReaderId;
    name: string;
    access: CatReaderAccess;
  };
  summary: string;
  symbols: string[];
  emotions: string[];
  themes: string[];
  interpretation: string;
  smallPrescription: string;
  readerNote?: string;
  tarot?: {
    cardName: string;
    symbol: string;
    dreamConnection: string;
    reflectionQuestion: string;
  };
};
```

MVP에서는 `tarot`를 실제 생성하지 않고, 회색고양이 클릭 이벤트와 준비 중 화면만 구현해도 된다.

## 프롬프트 분기 기준

모든 고양이는 공통 안전 원칙을 따른다.

공통:

- 참고 백과사전 항목을 사용한다.
- 진단, 예언, 질병, 불행 단정을 금지한다.
- "연결되어 보인다", "비유적으로 읽을 수 있다"처럼 가능성 기반으로 말한다.
- 고양이 말투는 낮은 강도로 제한한다.

고양이별 추가 지시:

| 고양이 | 프롬프트 추가 지시 |
| --- | --- |
| 검은냥 | 꿈속 상징과 백과사전 근거를 중심으로 해석한다. |
| 하얀냥 | 사용자가 느낀 감정을 부드럽게 이름 붙이고 안정을 돕는다. |
| 치즈냥 | 오늘 바로 할 수 있는 작은 행동을 구체적으로 제안한다. |
| 회색냥 | 최근 꿈 기록의 반복 패턴과 타로 상징을 함께 연결한다. 예언하지 않는다. |

## 분석/추천 로직

사용자가 직접 고양이를 선택할 수 있게 하되, 앱이 추천도 제공한다.

추천 규칙 초안:

| 조건 | 추천 |
| --- | --- |
| 첫 해몽 또는 선택 없음 | 검은고양이 |
| wakeMood가 불안, 무서움, 흐릿함 | 하얀고양이 |
| 꿈에 준비, 지각, 이동, 선택, 찾기 상징이 있음 | 치즈냥 |
| 같은 상징이 3회 이상 반복되거나 월간 정산 맥락 | 회색냥 |

MVP에서는 추천 로직을 단순하게 시작한다.

- 기본값은 검은고양이
- 감정이 불안이면 하얀고양이 추천 배지
- 결과의 작은 처방 CTA에는 치즈냥 추천
- 기록이 3개 이상이면 회색냥 잠금 카드 노출

## 측정 이벤트

| 이벤트 | 트리거 | 확인할 것 |
| --- | --- | --- |
| `cat_reader_selector_viewed` | 해몽사 선택 영역 노출 | 사용자가 고양이 차이를 볼 기회가 있었는가 |
| `cat_reader_selected` | 무료 고양이 선택 | 어떤 관점이 선택되는가 |
| `cat_reader_reinterpret_clicked` | 다른 무료 고양이로 다시 읽기 클릭 | 관점 차이에 관심이 있는가 |
| `gray_cat_locked_viewed` | 회색고양이 잠금 카드 노출 | 프리미엄 제안 노출 수 |
| `gray_cat_interest_clicked` | 회색고양이 관심 버튼 클릭 | 연간 프리미엄 의향 |
| `tarot_premium_interest_clicked` | 타로 리딩 CTA 클릭 | 꿈+타로 번들 수요 |

MVP 판단 기준:

- 무료 고양이 선택률이 30% 이상이면 고양이별 관점이 이해되고 있는 것이다.
- 다른 고양이로 다시 읽기 클릭률이 15% 이상이면 관점 재해석 기능 가치가 있다.
- 회색고양이 관심 클릭률이 5% 이상이면 프리미엄 후보로 유지한다.

## 구현 우선순위

### MVP

1. `catReaders` seed 데이터 추가
2. 꿈 입력 또는 결과 화면에 해몽사 선택 UI 추가
3. 검은냥/하얀냥/치즈냥 무료 선택 가능
4. 회색냥 잠금 카드와 관심 버튼 추가
5. 분석 요청에 `catReaderType` 전달
6. mock analysis에서 고양이별 `readerNote` 또는 작은 문체 차이 적용
7. 관련 이벤트 로깅 추가

### Post-MVP

1. 무료 고양이별 결과 섹션 차별화 강화
2. 같은 꿈을 다른 고양이로 짧게 다시 읽기
3. 회색고양이용 반복 상징/감정 패턴 계산
4. 타로 상징 리딩 구조 추가
5. 연간 Moon Pass와 회색고양이 연결
6. 월간 꿈 정산서에 회색고양이 코멘트 추가

## 완료 기준

- 사용자가 무료 고양이 3마리의 차이를 선택 화면에서 이해할 수 있다.
- 기본 해몽은 검은고양이로 자연스럽게 시작된다.
- 하얀냥과 치즈냥은 무료로 선택 가능하다.
- 회색냥은 잠겨 있지만 무엇이 특별한지 설명된다.
- 회색냥 CTA는 결제 대신 관심 이벤트를 남긴다.
- 결과 문구는 고양이별로 관점이 다르지만 안전 원칙을 위반하지 않는다.
