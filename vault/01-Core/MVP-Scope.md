# MVP Scope

> 1차 MVP는 “예쁜 기능이 많은 앱”이 아니라, 꿈 입력부터 기록 저장까지 끊기지 않는 앱이어야 한다.

---

## 검증할 질문

1. 사용자가 꿈을 입력할 의향이 있는가?
2. AI 해몽 결과가 흥미로운가?
3. 꿈 카드나 꿈 영수증이 저장하고 싶을 만큼 매력적인가?
4. 백과사전 기반 해석이 신뢰감을 주는가?
5. 다음날 다시 접속할 이유가 생기는가?

## 1차 MVP 필수 범위

| 우선순위 | 기능 | 완료 기준 |
| --- | --- | --- |
| P0 | 홈 화면 | 꿈 입력과 기억 안 남 흐름으로 바로 진입 가능 |
| P0 | 꿈 입력 | 꿈 원문, 날짜, 기분을 입력 가능 |
| P0 | 기억 안 남/아침 기분 기록 | 꿈이 없어도 기록 생성 가능 |
| P0 | LLM 구조화 분석 | 요약, 상징, 감정, 테마 JSON 생성 |
| P0 | 백과사전 샘플 DB | 최소 10개, 목표 30개 상징 seed |
| P0 | 백과사전 기반 해몽 | 참고 항목이 결과에 반영됨 |
| P0 | 결과 화면 | 요약, 상징, 해몽, 작은 처방 표시 |
| P0 | 꿈 카드/영수증 | 템플릿 기반 텍스트 결과물 생성 |
| P0 | 기록/꿈 달력 | 저장된 꿈을 날짜 기준으로 조회 |
| P1 | 상징 백과 전체/상세 | 백과사전 탐색 가능 |
| P1 | 공유 이미지 | 결과를 이미지로 저장/공유 |

## 1차 MVP 제외

- 이미지 생성 모델 기반 카드 생성
- 복잡한 카드 커스터마이징
- 커뮤니티
- 유료 결제
- 고급 리포트
- 임베딩 기반 유사 꿈 검색
- 많은 고양이 캐릭터
- 자기 전 밤의 기록
- 고양이방 고도화

## MVP 루프

```text
홈 진입
→ 꿈 입력 또는 기억 안 남 선택
→ 해석 또는 아침 기록 생성
→ 꿈 카드/꿈 영수증 확인
→ 기록 화면에 저장
```
