# Tarot Question-Based One Card

> 사용자가 지금 궁금한 상태를 고르고, 그 상태에 맞는 질문으로 한 장 타로를 여는 가벼운 리딩 흐름.

---

## 목적

질문형 한 장 타로는 기존 오늘의 타로보다 더 자주 열 수 있는 짧은 콘텐츠다. 사용자가 긴 질문을 직접 쓰지 않아도, 지금의 상태를 고른 뒤 앱이 준비한 질문 중 하나를 선택해 바로 한 장을 뽑게 한다.

핵심 포지셔닝은 예언이 아니라 체크인이다.

```text
마냥 타로는 거창한 예언보다, 오늘의 마음과 선택을 가볍게 비춰보는 카드 리딩.
```

## 기본 플로우

```text
타로 진입
→ "지금 어떤 게 궁금해?" 상태 선택
→ 상태별 질문 5개 중 하나 선택
→ 한 장 뽑기
→ 카드, 짧은 리딩, 오늘 볼 포인트 확인
→ 저장 또는 공유
```

## 첫 화면 상태 카드

첫 화면은 질문을 바로 나열하지 않고, 사용자가 자신의 현재 상태를 고르게 한다. 상태 문장은 기능 분류처럼 보이지 않아야 하며, 사용자가 자기 마음을 고르는 느낌이어야 한다.

| key | 화면 문구 | 대표 질문 |
| --- | --- | --- |
| `mind_complex` | 마음이 복잡해 | 오늘 내 마음이 붙잡고 있는 건 뭐야? |
| `relationship_concern` | 관계가 신경 쓰여 | 지금 이 관계에서 내가 봐야 할 마음은? |
| `work_blocked` | 일이 막힌 느낌이야 | 지금 이 일에서 가장 중요한 흐름은? |
| `reality_anxiety` | 돈이나 현실이 불안해 | 지금 내 현실에서 점검해야 할 건 뭐야? |
| `decision_point` | 선택을 앞두고 있어 | 이 선택에서 내가 진짜 봐야 할 기준은? |
| `daily_signal` | 그냥 오늘의 신호가 궁금해 | 오늘 내가 놓치지 말아야 할 신호는? |

## 질문 선택 화면

상태를 고르면 해당 상태에 맞는 질문 5개를 보여준다. 3개는 빈약해 보이고, 7개 이상은 선택 피로가 생기므로 5개를 기본값으로 둔다.

질문 원본은 [[Tarot-Question-Prompts]]에서 관리한다.

## 리딩 결과 구성

결과 화면은 짧고 바로 읽히는 구조를 우선한다.

| 영역 | 내용 |
| --- | --- |
| 선택 질문 | 사용자가 고른 질문 문장 |
| 카드 | 카드 이미지, 카드명, 정방향/역방향 |
| 짧은 리딩 | 질문 맥락에 맞춘 1장 해석 |
| 오늘 볼 포인트 | 한 문장 조언이 아니라, 오늘 확인할 기준 |
| 저장/공유 | 오늘 카드 기록, 결과 이미지 공유 |

## 덱 기준

질문형 한 장 타로는 78장 전체 덱과 잘 맞는다. 메이저만 쓰면 큰 전환과 운명적 상징이 과하게 자주 나오지만, 마이너를 포함하면 관계의 작은 신호, 일의 막힘, 돈과 생활 기반, 감정의 변화처럼 사용자가 매일 체감하는 리딩이 가능하다.

초기 실험 기준:

- 오늘의 한 장: 메이저 중심 유지 또는 78장 확장 실험
- 질문형 한 장: 78장 전체 덱 권장
- 3장 리딩: 심화 콘텐츠로 유지

## 무료/유료 연결

반복 타로 뽑기처럼 보이면 안 된다. 무료 사용자는 하루에 질문형 한 장을 열 수 있고, 더 깊게 보고 싶을 때 3장 리딩으로 이어지는 구조가 자연스럽다.

| 기능 | 권장 접근 |
| --- | --- |
| 오늘의 한 장 | 무료 |
| 질문형 한 장 | 무료 또는 일 1회 무료 |
| 같은 질문 다시 뽑기 | 제한 |
| 3장 리딩 | Moon Pass 또는 이벤트 |
| 주간 카드 회고 | Moon Pass 후보 |

## 측정 지표

- 상태 카드 선택률
- 질문 선택률
- 한 장 리딩 완료율
- 결과 저장률
- 공유 클릭률
- 3장 리딩 전환 클릭률
- 다음 날 재방문율

## 주의할 점

- 예/아니오 질문으로 만들지 않는다.
- 미래를 단정하지 않는다.
- 불안한 사용자를 반복 뽑기로 밀어붙이지 않는다.
- 질문 문장은 짧지만 기능 분류처럼 보이지 않게 쓴다.
- 결과는 "정답"보다 "오늘 볼 기준"으로 마무리한다.
